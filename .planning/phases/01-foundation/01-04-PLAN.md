   - Set default shortcuts matching current defaults: toggleMode = Shift+Option+Space, pushToTalk = Option+Space
   - Create a `HotkeyRecorderRow` SwiftUI view that takes a `title: String`, a `KeyboardShortcuts.Name`, and an `@ObservedObject var settingsStore: SettingsStore`
   - Inside HotkeyRecorderRow, use `KeyboardShortcuts.Recorder(for: name)` as the content
   - Add an `.onChange(of:)` modifier (or use `KeyboardShortcuts.onKeyUp` / observation) that syncs the recorded shortcut back to SettingsStore:
     - Read `KeyboardShortcuts.getShortcut(for: name)` to get the current shortcut
     - Convert shortcut.key?.rawValue to Int for keyCode (note: KeyboardShortcuts uses `Key` enum backed by Carbon virtual key codes which match CGEvent keyCode values)
     - Convert shortcut.modifiers (NSEvent.ModifierFlags) to CGEventFlags raw value for modifiers:
       - `.option` (NSEvent) = `.maskAlternate` (CGEventFlags) = 524288
       - `.shift` (NSEvent) = `.maskShift` (CGEventFlags) = 131072
       - `.command` (NSEvent) = `.maskCommand` (CGEventFlags) = 1048576
       - `.control` (NSEvent) = `.maskControl` (CGEventFlags) = 262144
     - Write the converted values to the appropriate SettingsStore properties

IMPORTANT: KeyboardShortcuts v2.x API — check the actual API. The Recorder view is `KeyboardShortcuts.Recorder(for:onChange:)` or `KeyboardShortcuts.Recorder(_ title:name:)`. Use whichever compiles. The key type may be `KeyboardShortcuts.Key` with a `.rawValue` that is the virtual key code (same as CGEvent keyboardEventKeycode).

Note: We use KeyboardShortcuts ONLY for the recorder UI. The actual hotkey monitoring stays on CGEventTap. The bridge writes recorded values to SettingsStore, and HotkeyMonitor reads from SettingsStore.
