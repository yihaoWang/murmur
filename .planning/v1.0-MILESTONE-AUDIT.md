## Integration Issues

### High Priority

1. **HOTKEY-03 — Modifier settings ignored at startup**
   - `startHotkeyMonitor()` sets only `keyCode`, never `modifiers`
   - `HotkeyMonitor.loadSettings(from:)` exists but is never called
   - Fix: call `hotkeyMonitor.loadSettings(from: settingsStore)` at startup and on settings change

2. **Confirm mode state corruption**
   - `recordingState` stuck in `.processing` if confirm sheet dismisses via MenuBarExtra focus loss
   - Fix: reset `recordingState = .idle` in the `Binding.set` closure for `pendingTranscription`

### Medium Priority

3. **LLM download progress invisible (LLM-03, UI-04)**
   - `appState.llmDownloadProgress` set but never rendered in `StatusItemView`
   - `llmDownloadProgress` never cleared to `nil` after completion
   - Fix: add `llmDownloadProgress` display to `StatusItemView`; clear after load

4. **Debug archiving skipped in confirm path (DEBUG-02)**
   - `DebugArchiver.save()` only in non-confirm branch; frames out of scope in callback
   - Fix: store frames in AppState alongside pendingTranscription

### Low Priority

5. **Dual hotkey system conflict risk**
   - `KeyboardShortcuts.Name` registered but no `onKeyDown` subscriptions
   - Both `KeyboardShortcuts` and `CGEventTap` intercept same keystrokes
   - Fix: remove unused `KeyboardShortcuts.Name` declarations or unify
